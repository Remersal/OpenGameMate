## 第 5 项成功标准

1. 测试前记录当前前台窗口句柄/进程与鼠标坐标。
2. ChatGPT WebView2 所在窗口保持非前台或隐藏测试状态。
3. 图片附件预览或等价的允许状态明确出现，固定测试文字进入输入区。
4. 用户确认后触发提交，且能从输入区/附件状态变化确认提交动作发生。
5. 测试后前台窗口与鼠标坐标未被 PoC 改变。
6. 全过程不读取回复正文、历史、Cookie、令牌或完整 HTML。

## 关键问题

1. 当前 ChatGPT 页面和目标账号是否允许在 WebView2/Voice 会话中添加图片？
2. WebView2 DevTools/DOM 能力能否在页面非焦点状态下安全设置文件输入并触发框架认可的输入事件？
3. 页面是否要求可信用户手势或可见/激活窗口才能提交？
4. Windows Graphics Capture 对当前系统、GPU 和全屏模式的实际表现如何？

## 已做决策

| 决策 | 理由 |
|------|------|
| Phase 0 是唯一实现范围 | 用户明确禁止一次性开发全部功能和自行扩展范围 |
| 最小 WebView2 外壳完成后先做第 5 项 | 用户明确要求优先验证，且文档将其列为发布阻断项 |
| 登录、麦克风和真实发送采用用户参与的手动验证 | PoC 不应获取凭据或在用户不知情时发送真实消息 |
| 只记录允许的最小状态，不保存网页内容 | 满足隐私边界和开发文档要求 |
| 失败时保留证据并转入报告，不继续完整产品建设 | 防止把理论可行误报为完成 |
| 将 .NET 8 SDK 隔离安装到仓库 `.dotnet/` 并忽略 | 本机无 SDK；该方式不修改系统 PATH，也不替换现有运行时 |
