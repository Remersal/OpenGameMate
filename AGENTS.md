# OpenGameMate 仓库工作约束

## 项目与当前阶段

- 技术基线：C#、.NET 8、WPF、Microsoft Edge WebView2、Windows Graphics Capture。
- 当前只允许完成 **Phase 0 技术可行性验证**。不得提前实现 v0.1.0 完整产品、系统托盘、定时任务、角色系统、远程适配规则、安装包、自动更新、商业化或其他未要求功能。
- 开发文档是范围依据：`docs/OpenGameMate_v0.1.0_开发文档.md`。若仓库中尚无该文件，应使用用户明确提供的原始文档，不得自行补写需求。
- 所有结论必须区分：自动验证、用户手动验证、未验证、失败、受环境限制。不得把推测、编译通过或页面元素存在写成真实功能成功。

## Phase 0 唯一目标

只验证并交付以下内容：

1. C# / WPF / .NET 解决方案基础结构。
2. WebView2 打开 ChatGPT 官方网页，并使用 OpenGameMate 独立用户数据目录。
3. 用户自行登录；用户手动验证网页麦克风权限和 Voice 可用性。
4. 捕获 Windows 主显示器，生成最大不超过 1920×1080 的临时截图，并验证清理行为。
5. 在不移动鼠标、不激活 ChatGPT 窗口、不抢占当前前台程序焦点的情况下，向后台 WebView2 中的 ChatGPT 页面添加测试图片与文字。
6. 触发一次用户明确发起的测试提交，并仅以允许的输入区/上传状态判断图片是否进入输入区域及提交是否被触发。
7. 不读取 ChatGPT 回复正文、聊天记录、Cookie、登录令牌、账号信息或网页完整 HTML。
8. 为每项验证记录环境、步骤、证据、成功/失败/未验证状态和限制条件。

## 强制优先级与停止条件

- 完成能承载实验的最小 WPF + WebView2 外壳后，**优先验证第 5 项**；不要先建设完整截图管线或产品功能。
- 第 5 项是发布阻断实验。若后台无焦点添加图片、文字或提交不可稳定实现：
  - 如实记录失败现象、复现步骤、前台焦点前后状态、使用的技术路径和根因判断；
  - 不得伪造、弱化或用“理论可行”替代成功结果；
  - 不得继续构建完整应用；
  - 只可完成独立、低成本的 Phase 0 证据收集和可接受替代方案评估。
- 不允许通过移动鼠标、模拟全局键鼠、激活或置顶 ChatGPT 窗口来冒充后台无焦点成功。
- 不绕过 ChatGPT 的验证、额度、权限或保护措施；不自动登录；不要求或保存用户凭据。

## 网页自动化与隐私边界

- 用户已明确要求 Phase 0 暂不实施域名限制，以优先验证登录及后台输入可行性。PoC 可导航到任意 HTTPS 主机，但仍须阻断无效 URI 与非 HTTPS 导航；日志只可记录主机和协议，不记录路径或查询参数。该诊断模式不得扩大其他隐私或自动化权限，真实登录仍由用户完成。
- 页面适配代码只允许：识别必要页面状态、定位输入控件、添加图片与文字、触发提交、检查上传/输入区状态和错误状态。
- 禁止提取、记录或持久化：回复文本、对话列表、聊天历史、页面完整 HTML、Cookie、Local Storage、Session Storage、授权头、Token、账号资料。
- 日志不得包含截图内容、用户输入正文、页面 HTML、凭据或带用户名的完整路径。选择器和最小布尔/枚举状态可以记录。
- 登录、麦克风授权、Voice 开启和真实消息发送必须由用户知情并参与。自动化测试不得在未明确提示的情况下向真实对话发送内容。
- 独立 WebView2 用户数据目录可以保留登录状态，但 PoC 必须清楚展示其位置和清理方式；不得读取其中的数据。

## 捕获与临时文件

- 只捕获 Windows 主显示器，不捕获其他显示器或枚举窗口内容用于分析。
- 输出图像宽高均不得超过 1920×1080，保持宽高比，不放大较小画面。
- 截图只写入 OpenGameMate 专用临时位置，用完立即删除；异常退出后的残留必须在下次启动或显式清理动作中处理。
- 不注入游戏、不读取游戏内存、不绕过反作弊或受保护内容；黑屏或拒绝访问应记为限制条件。

## 实现纪律

- 保持 PoC 最小化、可读、可手动复现；不要为了“以后可能需要”增加抽象或功能。
- 页面选择器和行为可能变化。代码必须失败关闭：无法确认目标元素时停止，不随机点击，不尝试相邻控件。
- 每个实验先定义可观察的成功标准，再执行，再把结果写入可行性报告。
- 编译、单元测试只能证明代码质量，不能替代真实 WebView2、登录、麦克风、前台焦点和上传实验。
- 不使用真实聊天回复作为测试证据；只允许验证输入区、附件预览、发送控件状态及可明确识别的上传错误。

## 文件与命令安全

- 禁止批量删除文件或目录。
- 不得使用 `del /s`、`rd /s`、`rmdir /s`、`Remove-Item -Recurse`、`rm -rf`。
- 如需删除，只能一次删除一个明确、已核对的文件路径，例如：`Remove-Item "C:\path\to\file.txt"`。
- 如果任务需要批量删除，立即停止并请用户手动处理。
- 保留用户已有改动和历史；不得执行 `git reset --hard` 或其他破坏性回退。

## Phase 0 交付标准

- 可运行的最小 PoC，而不是完整应用。
- 明确的启动方式和逐项手动测试步骤。
- 可行性报告逐项列出结果、证据与限制，尤其是第 5 项的焦点验证。
- 给出 Go / Conditional Go / No-Go 结论与可接受替代方案。
- 下一阶段建议只能基于 Phase 0 证据提出，不得在本阶段直接实现。
