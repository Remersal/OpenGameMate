# OpenGameMate v0.1.0 Test Plan

**Current status:** `Feature Complete, Pending User RC Validation`

This plan separates repeatable repository checks from user-controlled acceptance in a real game environment. Automated checks never sign in to ChatGPT, start Voice, capture a real desktop, send a real message, or replace the 30-minute and two-hour user soak runs. User acceptance must use non-private screen content.

## User-controlled RC acceptance

The user performs this section in the intended game, audio, account, display, and network environment. Record every result in [USER_RC_CHECKLIST_0.1.0.md](USER_RC_CHECKLIST_0.1.0.md). The user RC explicitly covers:

- continuous ChatGPT Voice use and Voice continuity around image updates;
- foreground-game and mouse-position preservation;
- 30-minute and two-hour stability runs;
- network interruption and recovery;
- ChatGPT page refresh and WebView2 window recovery;
- image-quota behavior when the account exposes that condition;
- application exit, restart, settings restoration, and temporary-file cleanup.

### Functional and safety acceptance

1. Start `OpenGameMate.App.exe`; verify only the main window opens and no screenshot is created.
2. Switch Chinese/English and restart; verify the setting persists but runtime state does not resume.
3. Open ChatGPT; verify the page is embedded in the right side of the OpenGameMate window, uses the isolated application WebView2 profile, and allows user-controlled sign-in.
4. Start Voice and confirm it in the main window. Verify OpenGameMate never displays or logs account data or audio.
5. Optionally send the full role text. Confirm the text is submitted once and no reply is read by OpenGameMate.
6. Start companion mode and accept the primary-display warning. Use a non-private borderless/windowed test scene.
7. Wait 30 seconds for the first automatic attempt. Verify one image and the automatic prompt are submitted only after page/audio idle stabilization, without moving the cursor or changing the foreground game window. Verify the next attempt occurs two minutes later.
8. Trigger **Send now** near an automatic due time. Verify only one submission occurs and the automatic conflict is skipped.
9. Pause across a due time, then resume. Verify no catch-up or immediate retry occurs.
10. Close the embedded ChatGPT page while running. Verify companion activity stops safely, the in-shell placeholder returns, and reopening the page requires explicit Voice confirmation.
11. Simulate or observe an adapter mismatch. Verify `AdapterError`, no random click, and access to rule retry and diagnostic export.
12. If an image-quota selector is observed, verify `VoiceOnly` and no retry/new-chat bypass.
13. Stop and exit. Verify the temporary PNG is absent and a restart restores settings only.
14. Export diagnostics and inspect the ZIP: it must contain JSONL logs only, with no screenshot, prompt, reply, account, token, audio, or full user path.

### User soak acceptance

1. Run companion mode continuously for 30 minutes in a real game. Record memory, CPU, WebView2 stability, automatic occurrences, upload failures, Voice continuity, and foreground/mouse behavior.
2. Run companion mode continuously for two hours. Record the same observations and confirm that no duplicate scheduler, screenshot backlog, stale catch-up image, or unbounded resource growth appears.
3. The soak runs are user-owned RC evidence. They are intentionally not executed by automated CI or by repository-maintenance work.

### User exception acceptance

1. Disconnect and restore the network; verify failure is bounded and no random click or rapid retry occurs.
2. Refresh ChatGPT; verify the adapter fails closed until the expected page state returns and Voice is explicitly reconfirmed when required.
3. Close the embedded ChatGPT page; verify safe stop, in-shell placeholder restoration, and explicit Voice confirmation after the user reopens it.
4. Exit and restart OpenGameMate; verify settings restore, runtime state does not silently resume, and temporary screenshots are not retained.
5. If an actual image-quota condition is available, verify transition to `VoiceOnly` without bypass, new-chat workaround, or immediate retry. Mark the item `BLOCKED/NOT OBSERVED` rather than simulating account capabilities that are unavailable.

## Targeted Voice-tail acceptance gate

This smaller gate must pass before starting either soak run:

1. Keep Voice speaking across an automatic due time. Verify OpenGameMate does not capture or touch the composer while WebView2 reports audio output.
2. Verify capture and composer preparation begin only after the page is structurally safe, WebView2 audio has remained silent continuously for at least three seconds, and page/audio state has remained stable for six seconds.
3. Verify the prepared attachment and prompt are submitted only after the second exact page/audio check; no stop-button, coordinate click, broad arrow match, or random control search is allowed.
4. Observe three consecutive successful automatic sends. Verify each uses a newly captured image and that no duplicate timer or screenshot backlog appears.
5. Keep Voice busy for longer than 90 seconds after a due time. Verify that PendingSend expires, the occurrence is skipped, and no stale screenshot is sent later.
6. Verify the foreground game window and mouse position remain unchanged throughout capture, attachment, text insertion, and submit.

## Targeted user-controlled live evidence already completed

- User-controlled sign-in, microphone permission, image attachment, text insertion, and real submit succeeded without OpenGameMate reading a reply, history, cookie, token, or audio content.
- Capture-only, attachment-only, and attachment-plus-text checks did not interrupt Voice. The actual interruption risk was isolated to submit during a Voice playback tail.
- The first 30-second occurrence and later two-minute occurrences succeeded. During active Voice output/activity, PendingSend deferred composer work; one continuously busy occurrence expired after 90 seconds without a stale catch-up image.
- Automatic submission retained the foreground application and mouse position, and the strengthened automatic prompt produced a natural proactive Voice topic from the latest image.
- These targeted results close the P0 experience gate but do not replace the user-owned 30-minute or two-hour RC soak runs or exception matrix.

## Compatibility limits

- Exclusive fullscreen, protected video, anti-cheat, or elevated games may return a black frame or fail. Use borderless/windowed mode; do not bypass protections.
- ChatGPT DOM structure, Voice/image availability, identity-provider redirects, quotas, and policies can change.
- Background image+text submission, Voice-safe deferral, expiry, and proactive image-topic behavior passed targeted live checks; long-duration stability still requires release-candidate soak testing.
