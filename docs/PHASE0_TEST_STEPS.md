# Phase 0 测试步骤

## 0. 准备

1. 按仓库根目录 `README.md` 构建并启动 PoC。
   - 如果登录弹窗顶部登录提供商持续转圈，先关闭 PoC，运行 `scripts\Install-Phase0FixedWebView2.ps1`，再重启。
2. 准备一个不含敏感内容的前台窗口，推荐记事本；也可以使用目标游戏。
3. 测试过程中不要让密码、令牌、私人聊天或其他敏感信息出现在截图中。

## 1. WebView2、独立用户数据、登录与麦克风

1. 点击“初始化并打开 ChatGPT”。
2. 确认出现标题为“OpenGameMate Phase 0 - ChatGPT”的独立窗口。
   - 最小证据日志应显示实际 WebView2 版本；本机隔离运行时实测为 `150.0.4078.65`。
3. 如果出现 Cloudflare 人机验证，由用户本人完成；PoC 不自动处理验证码。
4. 由用户本人登录 ChatGPT。PoC 不读取账号、Cookie 或登录令牌。
5. 登录完成后回到主窗口，点击“我已自行登录（记录人工通过）”。
6. 在 ChatGPT 页面由用户本人打开 Voice，并在浏览器权限提示中决定是否允许麦克风。
7. 确认可正常使用麦克风后，点击“我已验证 Voice 麦克风（记录人工通过）”。
8. 正常结束 Voice 时，使用 ChatGPT Voice 界面的退出控件；麦克风控件只负责静音/取消静音。
9. 如果网页退出控件无效，回到 PoC 点击“关闭 ChatGPT（结束语音）”。该操作会销毁当前 WebView2 会话并释放麦克风；再次点击“初始化并打开 ChatGPT”可重新创建窗口，独立用户数据目录和登录状态不会因此删除。

预期：用户数据只写入 `%LocalAppData%\OpenGameMate\Phase0\UserData`。当前按用户要求启用临时 Phase 0 HTTPS 导航诊断模式，不限制主机；无效 URI 与非 HTTPS 导航仍被阻断，日志只记录主机和协议。真实登录仍由用户自行完成。

后台输入实验点击后，ChatGPT 窗口会立即隐藏，这是为了验证真正的后台无焦点行为。倒计时结束并出现结果日志后，由用户手动点击“显示 ChatGPT”检查输入区；PoC 不会自动重新显示，以免抢回前台焦点并污染实验结果。

注意：“隐藏 ChatGPT”只改变窗口可见性，不结束 Voice，也不释放麦克风。需要结束语音时应使用网页退出控件或 PoC 的真正关闭按钮。

## 2. 主显示器截图

1. 点击“捕获主显示器”。
2. 检查界面显示的源尺寸、输出尺寸、字节数和固定临时路径。
3. 确认输出宽不超过 1920、高不超过 1080，且未被拉伸。
4. 可点击“删除临时截图”验证单文件清理；正常退出 PoC 也会清理当前临时截图。

已在本机实测：2560×1600 → 1728×1080，PNG 978,113 字节。

限制：独占全屏、受 DRM/系统保护内容、多 GPU、多显示器切换和 HDR 尚未覆盖；黑屏或受保护区域应记录为限制，不能伪报成功。

## 3. 最高优先级：后台无焦点加入图片和文字

此步骤会把一张由 PoC 生成的无隐私测试图及界面显示的固定英文文字加入当前 ChatGPT 输入区，但不会发送。

1. 先确认 ChatGPT 已登录，且当前是允许测试的普通对话输入区。
2. 勾选“我同意把无隐私测试图片和固定测试文字加入当前 ChatGPT 输入区（不会发送）”。
3. 点击“5 秒后后台加入图片和文字”。PoC 会隐藏 ChatGPT 窗口。
4. 立即仅用 `Alt+Tab` 切到记事本或游戏，避免移动鼠标；等待至少 6 秒。
5. 回到 PoC 查看结果。只有以下条件全部满足才算自动检查通过：
   - 图片文件被设置到页面文件输入控件，或图片粘贴路径产生了输入区附件预览；
   - 固定测试文字与输入区内容完全一致；
   - 测试前后的前台窗口进程、窗口句柄和鼠标坐标一致。
6. 点击“显示 ChatGPT”，由用户目视确认输入区确有图片预览和固定文字。
7. 目视通过后点击“我已看到输入区图片预览（记录人工通过）”。

如果页面结构发生变化、找不到唯一输入区、文件输入与粘贴路径均未让图片进入输入区、前台仍是 PoC/WebView2、鼠标位置变化或附件预览无法由用户确认，实验必须失败并停止，不会自动重试。

## 4. 用户明确确认后的单次提交

此步骤会向当前真实 ChatGPT 对话发送测试图片和固定文字。

1. 必须先完成步骤 3，并由用户目视确认输入区内容正确。
2. 勾选“我确认发送当前测试图片和文字”。
3. 点击“5 秒后后台触发一次提交”。
4. 立即仅用 `Alt+Tab` 切回记事本或游戏，等待至少 6 秒。
5. 回到 PoC 查看结果。PoC 只检查输入区或附件是否被清空等提交动作信号，不读取 ChatGPT 回复。

提交只执行一次，不自动重试。若按钮不存在、不是唯一按钮、处于禁用状态，或焦点/鼠标证据不稳定，则记为失败。

## 5. 证据核对

检查 `%LocalAppData%\OpenGameMate\Phase0\phase0-results.jsonl`。每行是一个 JSON 对象，只应包含：

- 时间；
- 检查项标识；
- Passed / Failed / Blocked / NotTested 状态；
- 不含网页内容的限制说明。

如果日志出现回复正文、聊天标题、Cookie、令牌或账号信息，应立即停止测试并视为实现缺陷。
